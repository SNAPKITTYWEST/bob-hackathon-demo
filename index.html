What It Does

1. Robot Submits

UiPath Robot sends contracts, invoices, claims, reports, or exception packets into BOB for compliance reasoning.

2. BOB Reasons

BOB asks Claude Sonnet 4.6 through AWS Bedrock for grounded compliance analysis against policy and evidence.

3. WORM Seals

EVIDENCE allows execution. SILENCE blocks action. The decision is SHA-256 sealed into an append-only audit trail.

Execution Flow

Input

UiPath submits a document bundle and a requested action: approve, escalate, pay, file, route, or stop.

Reason

BOB invokes Claude Sonnet 4.6 on AWS Bedrock with policy context and refuses unsupported claims.

Verdict

EVIDENCE means the robot may continue. SILENCE means the robot must stop and escalate to a human.

Seal

Every verdict becomes a SHA-256 WORM receipt: payload hash, policy hash, timestamp, and process ID.

Formal Theorems

THEOREM: Verdict Completeness

For every UiPath document packet P
and every requested action A:

BOB(P, A) returns exactly one verdict:

  EVIDENCE  -> action may proceed
  SILENCE   -> action is blocked

No ambiguous, partial, or speculative
verdict is valid.
THEOREM: WORM Integrity

For every verdict V emitted by BOB:

seal = SHA256(
  packet_hash,
  policy_hash,
  verdict,
  timestamp,
  previous_seal
)

Any mutation changes the seal.
Therefore the audit chain is tamper-evident.
THEOREM: Trust Deed Soundness

If Trust Deed policy D forbids action A,
then no model output can authorize A.

Model reasoning is advisory.
Trust Deed constraints are binding.

Therefore policy cannot be bypassed
by fluent language.
THEOREM: Human-in-Loop Guarantee

If BOB returns SILENCE,
UiPath execution halts.

The packet is sealed,
the reason is logged,
and the workflow enters human review.

Therefore unsupported automation
cannot silently continue.
